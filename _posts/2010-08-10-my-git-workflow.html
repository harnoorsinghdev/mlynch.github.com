I've recently settled on a Git workflow that works pretty well for me:

For initial project creation:

  1. Go to remote server
    1. Create a folder myproject.git
    2. cd into folder, run  git --bare init
  2. Go to local machine
    1. mkdir for project. Ex: mkdir myproject
    2. cd into that directory. Run git remote add origin ssh://me@myserver.com:SSHPORT/path/to/myproject.git
    3. Add files to the repo. git add doc1 doc2 etc
    4. git commit -m "Initial import"
    5. git push origin master

For branching, merging, daily work, etc., I tend to work like this:

  1. Get off master branch. Run git checkout -b mybranch
  2. If new branch, run git push origin mybranch

Right now I have three "stages" of GoBuzz deployment: dev, stage and prod. I'll commit changes in dev, go to stage, and run git pull origin dev, fixing any merge conflicts if necessary.

Grabbing the code on another machine is easy by doing git clone ssh://me@myserver.com:SSHPORT/path/to/myproject.git and then git checkout -b mybranch and git reset --hard origin/mybranch to sync the local branch with the master branch.
